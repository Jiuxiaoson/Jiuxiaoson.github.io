UiAutomator初学过程记录

准备工具

1.安装Eclipse,并安装ADT插件

2.JDK1.8,SDK(API18)

3.ANT(1.10.1)

环境搭建

1.右键我的电脑—->属性—->高级系统设置—->环境变量

2.在系统变量里面新建JAVA_HOME和ANT_HOME,JAVA_HOME是jdk里面的bin的上级目录,ANT_HOME是ant里面的bin的上级目录

3.在path中添加%JAVA_HOME%\bin,再添加%ANT_HOME%\bin

4.打开dos命令行,输入java测试java配置是否生效,输入ant测试ant环境配置是否生效

详细过程

1.建立工程,利用Eclipse新建一个java project

2.添加JUnit库

3.添加android库

​ 需要导入两个jar包，在android-sdk/platforms/android-18目录下的android.jar和uiautomator.jar两个包

4.写一个测试java文件

5.找sdk的id

​ 注意,这个需要dos切换到Android-sdk\tools目录下,输入android list,找到前面添加库的那个版本（18）,前面是sdk的id

6.创建build文件

​ 这个还是Android-sdk\tools目录下,输入android create uitest-project -n (输入名称) -t （sdk的id） -p （项目路径）

7.编译生产jar

​ 这个需要切换到项目目录下，输入ant build命令，注意路径

8.导入设备

​ 命令adb push data/local/tmp，这个要注意路径要切换到bin下面，因为你生产的jar是在bin下面

9.运行脚本

​ adb shell uiautomator runtest -c <工程中的类的全路径>

对于UiAutomator的API统计

ISSUES

adb shell 出现unknow host service 错误

原因是：5037端口被占用，这个端口是连接手机用的，被占用之后adb命令就连接不上手机

解决办法：dos命令netstat -aon | findstr “5037”查找到占用该端口的PID，然后打开任务管理器，将该进程手动杀死
